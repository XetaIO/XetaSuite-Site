Ce guide vous accompagne dans l'installation du backend Laravel de XetaSuite.

## Prérequis

- **PHP** 8.2+ (8.3 ou 8.4 recommandé)
- **Composer** 2.x
- **PostgreSQL** 16, 17 ou 18
- **Node.js** 20+ (pour les assets)
- **Redis** (optionnel, pour les queues)

## Installation

<Steps>

1. **Cloner le repository**

   ```bash
   git clone https://github.com/xetaio/XetaSuite-core.git
   cd XetaSuite-core
   ```

2. **Installer les dépendances PHP**

   ```bash
   composer install
   ```

3. **Configurer l'environnement**

   ```bash
   cp .env.example .env
   php artisan key:generate
   ```

4. **Configurer la base de données**

   Éditez le fichier `.env` :

   ```env
   DB_CONNECTION=pgsql
   DB_HOST=127.0.0.1
   DB_PORT=5432
   DB_DATABASE=xetasuite
   DB_USERNAME=your_user
   DB_PASSWORD=your_password
   ```

5. **Exécuter les migrations**

   ```bash
   php artisan migrate --seed
   ```

6. **Lancer le serveur**

   ```bash
   php artisan serve
   ```

</Steps>

## Configuration Sanctum

<Aside type="tip">
  XetaSuite utilise **Sanctum stateful authentication** (cookies) pour l'authentification SPA.
</Aside>

Configurez les domaines autorisés dans `.env` :

```env
SANCTUM_STATEFUL_DOMAINS=localhost:5173,xetasuite.test
SESSION_DOMAIN=.xetasuite.test
```

## Structure du projet

```
app/
├── Actions/          # Classes d'action single-purpose
├── Http/
│   ├── Controllers/Api/V1/   # Contrôleurs API
│   ├── Requests/V1/          # Form Requests
│   └── Resources/V1/         # API Resources
├── Models/           # Modèles Eloquent
├── Observers/        # Observers pour les événements modèle
├── Policies/         # Policies d'autorisation
└── Services/         # Services métier
```
